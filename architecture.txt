=============  МОДЕЛЬ ДАННЫХ  =============

1. Получение данных от пользователя

Запрос любых данных повторяется до получения корректных данных.
    
    команды главного меню
        
                              начать новую партию : new    : n : начать   : н :
                    загрузить существующую партию : load   : l : загрузка : з :
                         отобразить раздел помощи : help   : h : помощь   : п :
              создать или переключиться на игрока : player : p : игрок    : и :
                   отобразить таблицу результатов : table  : t : таблица  : т :
                             изменить размер поля : dim    : d : размер   : р :
                                            выйти : quit   : q : выход    : в :
    
    ход игрока
        
        цифровые символы
        номер свободной клетки
    
    имя игрока
        
        ... (используемые символы?)
        уникальность



2. Структуры данных
    
    статистика игроков
        
        {
          'игрок_1': {'побед': 2, 'поражений': 3, 'ничьих': 4},
          'игрок_2': {'побед': 0, 'поражений': 0, 'ничьих': 0},
        }
        Сохраняется в файле players.ini
    
    сохранённые партии
    
        {
            'партия_1': {
                'игрок_1': 'имя_1',
                'игрок_2': 'имя_2',
                'ходы': (структура данных ходов),
                'результат': 'имя_победителя | ничья | None (не завершено)',
            }
        }
        Сохраняется в файле games.ini
        
        
        
        
    
    сделанные ходы                                   1 | 2 | 3 
                                                    ———————————
        {5: 'X', 3: 'O', 9: 'X'}                     4 | 5 | 6 
                                                    ———————————
        [5, 3, 9]                                    7 | 8 | 9 
    
    выигрышные комбинации
        
        [
          {1, 2, 3},
          {4, 5, 6},
          {7, 8, 9},
          {1, 4, 6},
          {2, 5, 8},
          {3, 6, 9},
          {1, 5, 9},
          {3, 5, 7},
        ]
        


3. Файлы данных
    
    статистика игроков — players.ini
        
        [игрок_1]
        побед = 2
        поражений = 3
        ничьих = 4
        
        [игрок_2]
        побед = 0
        поражений = 0
        ничьих = 0
    
    сохранённые партии - games.ini
    
        [id_партии]
        игрок_1 = имя_1
        игрок_2 = имя_2
        ходы = (структура данных ходов)
        результат = имя_побелителя | ничья
        завершено = True | False
        

=====  ПОСЛЕДОВАТЕЛЬНОСТЬ ВЫПОЛНЕНИЯ  =====


ГЛАВНОЕ МЕНЮ

ИГРОКИ
    Получаем игроков
    Получаем имена игроков для игры
    Проверяем наличие игроков в списке всех игроков
    Если если кого-то нет, то создаем с новым именем
    
ИГРА
    Проверяем наличие неоконченной партии для игроков
    Запуск новой игры или загрузка существующей

ХОДЫ
    Загрузка ходов если загружена не оконченная игра
    Подсчет ходов, продолжение подсчета ходов

ПОБЕДА
    Проверка после каждого хода на победу игрока сделавшего ход
    Проверка на ничью из оставшихся комбинаций

РЕЗУЛЬТАТЫ
    Обновление данных игравших игроков в файле players.ini
    Обновление файла games.ini с внесением новой партии, либо изменения существующей

ПОВТОРНЫЙ ЗАПУСК
    Сообщение об окончании игры
    Продолжить | Показать результаты | Выход
