=========  МОДЕЛЬ РАСПРОСТРАНЕНИЯ  ========

Файл, точка входа для интерпретатора Python, вспомогательные файлы с кодом.

    xox.py | exe.py | main.py - точка входа
    
    players.py - функции обработки данных об игроках
    menu.py - функции обработки команд меню
    xprint.py - функции для форматированного вывода данных в консоль
    
    temp.py - временный файл для запуска и отладки
    utils.py - временный файл для хранения прочих используемых функций
    
Файлы данных.

    players.ini - хранение данных об игроках
    games.??? - хранение данных об играх


=============  МОДЕЛЬ ДАННЫХ  =============

1. Получение данных от пользователя

Запрос любых данных повторяется до получения корректных данных.
    
    команды главного меню
        
                              начать новую партию : new    : n : начать   : н :
                    загрузить существующую партию : load   : l : загрузка : з :
                         отобразить раздел помощи : help   : h : помощь   : п :
              создать или переключиться на игрока : player : p : игрок    : и :
                   отобразить таблицу результатов : table  : t : таблица  : т :
                             изменить размер поля : dim    : d : размер   : р :
                                            выйти : quit   : q : выход    : в :
    
    ход игрока
        
        цифровые символы
        номер свободной клетки
    
    имя игрока
        
        ... (используемые символы?)
        уникальность



2. Структуры данных

    структура меню (команды главного меню):
    
        [
            ('начать новую партию', 'new', 'n', 'начать', 'н'),
            ('загрузить существующую партию', 'load', 'l', 'загрузка', 'з'),
            ('отобразить раздел помощи', 'help', 'h', 'помощь', 'п'),
            ('создать или переключиться на игрока', 'player', 'p', 'игрок', 'и'),
            ('отобразить таблицу результатов', 'table', 't', 'таблица', 'т'), 
            ('изменить размер поля', 'dim', 'd', 'размер', 'р'),
            ('выйти', 'quit', 'q', 'выход', 'в')
        ]
    
    статистика игроков (сохраняется в файле players.ini)
        
        {
          'игрок_1': {'побед': 2, 'поражений': 3, 'ничьих': 4},
          'игрок_2': {'побед': 0, 'поражений': 0, 'ничьих': 0},
        }
    
    сохранённые партии (сохраняется в файле saves.ttt)
    
        {
            ('player_01', 'player_02'): ([1, 2, 4, 3, 9], ['X', '0', '0', 'X', '_', '_', '_', '_', 'X']),
            ('player_01', 'player_02'): ([5, 3, 9, 8, 1], ['X', '0', '_', '_', 'X', '_', '_', '0', 'X']),
        }
            
        
    символы подстановки в шаблон 
    
        ('O', 'X')
    
    элементы подстановки в шаблон (состояние заполненного поля)
    
        [' ', ' ', 'X', ' ', 'O', ' ', ' ', 'X', ' ', ]
    
    игровое поле и символы клавиатуры для хода игрока
    
         1 | 2 | 3 
        ———————————
         4 | 5 | 6 
        ———————————
         7 | 8 | 9     
    
    история сделанных ходов 
    
        [5, 3, 9, 1, 8]
    
    выигрышные комбинации
        
        [
          {1, 2, 3},
          {4, 5, 6},
          {7, 8, 9},
          {1, 4, 6},
          {2, 5, 8},
          {3, 6, 9},
          {1, 5, 9},
          {3, 5, 7},
        ]
        


3. Файлы данных
    
    статистика игроков — players.ini
        
        [игрок_1]
        побед = 2

        ничьих = 4
        
        [игрок_2]

=====  ПОСЛЕДОВАТЕЛЬНОСТЬ ВЫПОЛНЕНИЯ  =====

НАЧАЛО
    Показать название игры
        Вывод в консоль "Крестики-Нолики"
    Показать описание игры
        Вывод в консоль - "Описание игры и правила"
    Показать help
        Вывод в консоль - таблица "Команды меню"
    

ГЛАВНОЕ МЕНЮ
    ПОЛУЧАЕМ ВСЕХ ИГРОКОВ
        Чтение файла players.ini
    
    ПОЛУЧАЕМ ИГРОКА_01
        Запранниваем имя_1
        Проверяем наличие игрока_1 в списке игроков
            Если нет - то добавляем
            Получаем всех игроков (обновляем)
    
    КОМАНДА ПОЛЬЗОВАТЕЛЯ (ЗАПРОС)
        HELP - Показать help
            Вывод в консоль - таблица "Команды меню"
        
        PLAYER - Создать игрока
            Проверяем наличие игрока_1 в списке игроков
                Если нет - то добавляем
                Получаем всех игроков (обновляем)
        
        LOAD - Загрузить игру
            Считываем файл с сохраненными играми
            Обрабатываем незавершенные игры
                Если у игрока есть незавершенные игры вывести список
                    Запрос номера (id) игры
                    Загрузка игры с восстановлением состояния
        NEW - Новая игра
            Режим игры
                ИГРА С БОТОМ
                    Уровень сложности
                ИГРА С ДРУГОМ
                    Получаем имя игрока_2
                    Проверяем наличие игрока_2 в списке игроков
                        Если нет - то добавляем
                        Получаем всех игроков
                    Выбор токена (O, X) первым игоком
                ИГРА ...
                    ХОДЫ
                        Загрузка ходов если загружена не оконченная игра
                        Подсчет ходов, продолжение подсчета ходов
                    ПОБЕДА
                        Проверка после каждого хода на победу игрока сделавшего ход
                        Проверка на ничью из оставшихся комбинаций
                    СТАТИСТИКА
                        Запись (обновление данных)
                            players.ini
                            games.ini
                    ПОВТОРНЫЙ ЗАПУСК
                        Сообщение об окончании игры
                        Продолжить | Показать результаты | Выход
        
        TABLE - Результаты (таблица)
            Вывод в консоль - таблица (отсортированная) со статистикой
            
        DIM - Изменить размер поля
        
        QUIT - Выход
